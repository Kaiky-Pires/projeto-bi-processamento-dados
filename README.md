# Processamento e Transformação de Dados com Power BI

# 1. Objetivo do Projeto
Este projeto detalha a etapa de ETL (Extract, Transform, Load), fundamental para garantir a integridade e a qualidade dos dados antes da criação de qualquer visual. O foco está na manipulação de dados brutos para a criação de um modelo de dados otimizado.

# 2. Etapas de Processamento Lógico (Power Query)
Para este desafio, a arquitetura de dados seguiu as seguintes diretrizes técnicas:
Limpeza de Dados: Identificação e tratamento de valores nulos ou inconsistentes.
Transformação de Tipos: Conversão de colunas numéricas e de texto para garantir que cálculos matemáticos (Soma, Média, Mediana) ocorram sem erros.

Criação de Colunas Condicionais: Lógica para categorização de dados financeiros, permitindo segmentações personalizadas.
Mesclagem e Agrupamento: Consolidação de tabelas relacionadas para simplificar o modelo de dados, reduzindo a complexidade para o usuário final.

# 3. Modelagem de Dados
O modelo foi planejado para suportar relações entre diferentes fontes de dados, garantindo que os filtros aplicados em uma tabela reflitam corretamente em todo o relatório. Isso inclui a organização de chaves primárias e estrangeiras dentro do ambiente de BI.

# 4. Justificativa Técnica
A entrega foca na documentação técnica dos processos de transformação. Devido à ausência de ambiente para execução do Power BI Desktop, este repositório serve como guia de implementação e prova de conhecimento dos conceitos de tratamento de dados e lógica de ETL.
